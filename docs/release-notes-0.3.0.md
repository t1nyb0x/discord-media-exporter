# Discord Media Exporter 0.3.0 リリースノート

## 概要

表示中かつユーザーが選択したDiscord添付を、一つのメディアZIPとして保存できるようになりました。従来の個別保存も引き続き利用できます。

## 追加機能

- 「ZIPにまとめて保存」操作
- ZIP取得・生成・保存の進捗とキャンセル
- 同名ファイルの安全な連番付与
- popupを閉じた後の処理継続と状態再表示
- 100件、1ファイル50 MiB、合計100 MiBのハード上限

## 権限変更

- `offscreen`: popupとは独立してBlob形式のZIPを生成するために追加
- `cdn.discordapp.com`, `media.discordapp.net`への任意ホスト権限: ZIP開始時だけChromeの確認を表示し、処理終了時に自動で解放

恒久的な必須host permission、Cookie、ユーザートークン、Discord内部APIは使用しません。権限を許可しない場合も個別保存は利用できます。

## 安全性と制限

- 全項目の取得に成功した場合だけZIPを保存し、不完全な部分ZIPは残しません
- CDN取得は一件ずつ行い、実際に読み込んだバイト数で上限を判定します
- redirect後もHTTPS、許可ホスト、添付パスを検証します
- Chrome終了、拡張機能のreload/update後は生成を再開できません

## 実機確認

2026-07-16にProject ownerから、メディアZIP出力を実機で確認済みとの報告を受けました。

Chromeバージョン、OS、25 / 50 / 100 MiBごとの測定値、個別チェック項目の結果は共有されていないため、本リリースノートでは推測して補完しません。
