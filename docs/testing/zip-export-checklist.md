# Phase 5 メディアZIP手動テスト

> [!NOTE]
> 2026-07-16にProject ownerからメディアZIP出力の実機確認済みとの報告を受け、Phase 5は完了扱いです。Chromeバージョン、OS、容量別測定値、以下の個別チェック結果は共有されていないため、推測でチェック済みにはしていません。

## 前提

- 最新のChrome Stableを使用する
- `pnpm release:prepare`が成功した`.output/chrome-mv3`をunpacked extensionとして読み込む
- 権利が明確なテスト用Discordチャンネルとテストメディアだけを使う
- DevToolsのMemory、Network、拡張機能service worker、offscreen documentを確認できるようにする

## 基本フロー

- [ ] 「自動収集を開始」を一度押すと、現在の表示範囲を収集できる
- [ ] popupを閉じてDiscordを手動スクロールすると、新しく表示した画像・動画・その他添付が追加操作なしで累積される
- [ ] 同じ添付を再表示しても候補が重複しない
- [ ] popupを再度開くと累積候補と「自動収集を停止」を表示できる
- [ ] 自動収集を停止した後のスクロールでは候補が追加されない
- [ ] 別チャンネルへ移動すると自動収集が停止し、以前のチャンネルの候補と混在しない
- [ ] 「収集をクリア」で候補を削除できる
- [ ] 任意の4件を選択して「ZIPにまとめて保存」を実行できる
- [ ] ZIP開始時だけDiscord CDN 2ホストへの権限確認が表示される
- [ ] 取得中ファイル名、件数、バイト数、ZIP作成、保存の状態が表示される
- [ ] 保存されたZIPを標準の展開機能で開ける
- [ ] 4 entryの名前と内容が選択項目に一致する
- [ ] ZIP名にチャンネル名、ユーザー名、メッセージ内容が含まれない
- [ ] 完了後にDiscord CDNの任意権限が解放される
- [ ] 完了後にoffscreen documentとBlob URLが残らない
- [ ] ZIP実行後も従来の個別保存が利用できる

## ファイル名

- [ ] 同名ファイルが`name.ext`, `name (2).ext`として両方格納される
- [ ] 大文字・小文字だけが異なる名前も衝突しない
- [ ] 日本語名を展開後も正しく表示できる
- [ ] 180文字へ切り詰めた場合も拡張子が残る
- [ ] パス区切り、制御文字、Windows予約名をZIP entryに使用しない

## 失敗とキャンセル

- [ ] 権限拒否時に取得を開始せず、個別保存は利用できる
- [ ] 403または期限切れURLを含む場合に部分ZIPを保存しない
- [ ] ネットワーク切断時にURLを含まない失敗理由を表示する
- [ ] 取得中にキャンセルでき、ZIPを保存しない
- [ ] キャンセル後に任意権限とoffscreen documentを解放する
- [ ] popupを閉じて再度開いた時に進行状態を表示する
- [ ] 拡張機能をreloadした場合に再開せず、安全な失敗として表示する

## 容量とメモリ

各ケースで開始前、最大、完了後のメモリ、所要時間、キャンセル応答時間を記録します。単一ファイルは50 MiB以下にし、複数ファイルで合計容量を作ります。

| 合計入力 | 開始前 | 最大 | 完了後 | 所要時間 | キャンセル応答 | 結果 |
| -------- | ------ | ---- | ------ | -------- | -------------- | ---- |
| 25 MiB   |        |      |        |          |                | [ ]  |
| 50 MiB   |        |      |        |          |                | [ ]  |
| 100 MiB  |        |      |        |          |                | [ ]  |

- [ ] 50 MiBを超える単一responseを`ITEM_TOO_LARGE`相当で拒否する
- [ ] 合計100 MiBを超えた時点で読み取りを停止し、部分ZIPを保存しない
- [ ] 100 MiB処理中もpopupとキャンセル操作が応答する
- [ ] 完了またはキャンセル後にメモリが継続的に増加しない

## 通信とログ

- [ ] 通信先が`cdn.discordapp.com`と`media.discordapp.net`だけである
- [ ] requestにCookieとAuthorization headerがない
- [ ] redirect後の許可外ホストを拒否する
- [ ] console、service worker状態、`chrome.storage.session`に完全なURLが残らない
- [ ] 外部ZIPサービス、分析、クラッシュ送信への通信がない
