## 実機課題

### COLLECTOR-STATUS-01: popup再表示時にボタンがOFF表示になる

- 再現: 「自動収集を開始」を押し、ON状態を確認して収集を継続する。一定時間後にpopupを再度開く。
- 期待結果: collectorが動作中であれば「自動収集を停止」と表示する。
- 実際の結果: 自動収集と候補追加は継続しているが、ボタンは「自動収集を開始」のOFF状態で表示される。
- 影響: 収集処理は継続するが、ユーザーが監視状態を正しく判断できない。
- 原因候補: 注入済みcollectorの`runtime.onMessage` listenerが同期オブジェクトを返していた。Chromeのmessage responseとして確実に返るPromiseではないため、popupが`undefined`を受け取ってOFFと判定する可能性があった。
- 修正: `0.4.1`でcollector statusと停止応答を、認識済みメッセージに対してだけ`Promise<CollectorResponse>`で返す。
- 自動回帰: popup再表示時のON復元と、entrypoint listenerがPromiseを返すことをテストへ追加。
- 実機回帰: 2026-07-16にProject ownerがPassを確認。

この問題は`0.4.1`で修正済みとして扱います。
