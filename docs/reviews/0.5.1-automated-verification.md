## 検証方法

4 GiB級の実データは生成せず、ZIP64 central directory extra field、ZIP64 end record、locatorへ渡す`bigint`のsize・offsetを境界値で直接検証します。65,535件と65,536件は実際に空entryをwriterへ渡し、ZIP64 end recordの64-bit entry countを確認します。

OPFS pipelineはhappy-dom上のFile System adapterを使用し、101件のresponseをZIP64 writerからOPFS writableへ流し、返された`File`を展開してentry数と内容を確認します。Chrome downloadsはadapter mockを使用し、`FILE_NO_SPACE`の状態遷移とcleanupを確認します。

## 自動検証の限界

この検証はZIP64構造とextension内の制御フローを対象とし、実Chromeプロセス、物理ディスク、OS標準展開機能は使用しません。したがって、実メモリ、実ディスク枯渇、4 GiB超の保存時間、OS固有の展開互換性は別の確認項目です。
