# Discord Media Exporter 0.6.0 リリースノート

## 概要

自動収集開始後のDiscord画面へ「ガイド付き収集」を表示し、ユーザーが押すたびに古い投稿方向へ一画面だけ遡れるようにしました。

無人で上端まで移動する機能ではありません。一回のclickに対して一回だけ表示高の80%を移動し、次の移動には再度ユーザー操作が必要です。

## 追加

- Discord画面右下のShadow DOMガイド
- 「1画面戻る」と「停止」
- message viewportからscrollable ancestorを特定する処理
- 上端、scroll container不明、500件到達時の状態表示
- 停止・チャンネル変更時のガイドcleanup
- 明示操作時だけ可視スポイラーを最大50件解除するbutton
- ZIP内ファイルを候補の取得順に`001_`、`002_`の連番で格納

## 修正

- OPFSから取得した完成`File`のMIME typeが空の場合も、`application/zip`のBlobとして保存する
- ZIP対象をcheckboxの選択順ではなく、候補registryへの取得順で処理する
- Chrome環境によって完成archiveが`.txt`として扱われる問題を修正する

## 維持する境界

- popupで自動収集を開始した後だけ動作
- 可視範囲に表示されたDiscord添付だけを収集
- 同じチャンネル内、最大500件
- 保存前に候補を確認・選択
- 無人の連続scroll、定期巡回、内部API、token、Cookieを不使用
- 必須permissionと任意host permissionの追加なし
- スポイラー解除は可視範囲、aria-label、最大50件に限定し、自動反復しない

## 規約上の注意

Discordの利用規約は、書面による同意のない自動的なscrapingを禁止しています。このガイド付き方式が公式な適用除外を受けることは確認できていません。利用前に規約、保存する権利、サーバールールを確認してください。

## 実機確認

- 実Discordで古い投稿方向へ移動すること
- 一回の操作で一回だけ移動すること
- 古い投稿の追加描画後も次操作で継続できること
- ガイドがDiscordの操作を妨げないこと
- 停止、チャンネル変更、reloadでガイドが残らないこと
- 表示中のスポイラーだけを解除し、解除後の画像を収集できること

Project ownerは2026-07-16にガイド付き収集の基本動作を確認しました。ChromeバージョンとOSは共有されていないため推測で補完しません。スポイラー解除、取得順の連番、`.zip`出力は追加修正後の再確認対象です。
