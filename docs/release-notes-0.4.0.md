# Discord Media Exporter 0.4.0 リリースノート

## 概要

Discordチャンネルで「自動収集を開始」を一度押した後、ユーザーが手動スクロールするだけで、画面に現れたメディア候補を自動的に追加できるようになりました。累積した候補は従来どおり、個別保存または一つのメディアZIPとして保存できます。

## 追加機能

- scroll、resize、Discord DOM変更を監視する可視範囲の自動収集
- 250 ms debounceによる再走査回数の抑制
- popupを閉じた後の収集継続と、再表示時の状態・候補復元
- 「自動収集を停止」操作
- 別チャンネル移動時の自動停止
- 同じチャンネルの候補を最大500件まで保持するセッション収集
- URL単位の重複排除と「収集をクリア」操作
- 一時的な候補登録失敗後の再試行

## 権限

`0.3.0`から必須権限と任意host permissionの追加はありません。自動収集は、ユーザーがツールバーから開始した対象タブへ`activeTab`と`scripting`で一時的に注入したスクリプト内で動作します。manifest常駐content scriptや恒久的なDiscord host permissionは使用しません。

## 安全性と制限

- 各走査では、その時点でメッセージ表示領域と交差する可視メディアだけを扱います
- 自動スクロール、定期巡回、複数チャンネルの無人収集は行いません
- Discord内部API、Gateway、Cookie、ユーザートークンを使用しません
- 保存は一覧でユーザーが明示選択した候補だけを対象とします
- 候補URLとファイル名はChromeセッション中または「収集をクリア」まで端末内だけに保持します
- 停止操作、別チャンネル移動、ページ再読み込み、タブ終了で自動収集を停止します
- DiscordのDOM変更により、検出や停止判定が影響を受ける可能性があります

## リリース前の残確認

- 実Discordで長時間スクロールした場合の候補追加、重複排除、CPU使用状況
- popup close/reopen、自動停止、明示停止、チャンネル移動、reload、タブ終了
- 500件上限到達時の表示と保存
- Project ownerから報告された自動収集の実機課題について、再現手順、環境、期待結果、実際の結果を記録
- 自動収集の課題修正後にChrome実機で回帰確認

Phase 5のメディアZIP出力は`0.3.0`で実機確認済みです。上記の自動収集課題が完了するまで、`0.4.0`を実機確認済みとして扱いません。
