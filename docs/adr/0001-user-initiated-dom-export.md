# ADR-0001: 表示中メディアの保存支援

- Status: Accepted
- Date: 2026-07-15
- Decision owners: Project owner

## Context

ブラウザで閲覧中の Discord チャンネルからメディアを保存する方法として、次の案があります。

1. 現在のメッセージ表示領域内にあるメディアをユーザー操作時に一覧化する
2. 通常ユーザーの認証情報で Discord の内部 API を呼ぶ
3. DOM を自動スクロールして履歴を収集する
4. 公式 Bot API を使い、Bot が参加・閲覧できる範囲をエクスポートする
5. ユーザーが一件ずつブラウザ標準機能で保存する

Discord の利用規約は書面による同意のないスクレイピングを禁止し、通常ユーザーアカウントの自動化も禁止されています。また、ブラウザ拡張は私的なメッセージや期限付き URL に接触し得るため、機能だけでなく権限とデータ境界を明確にする必要があります。

## Decision

MVP は案 1「現在のメッセージ表示領域内にあるメディアを、ユーザー操作時に一覧化する」を採用します。実装と利用は、以下のガードレールを満たす範囲に限定します。

- ユーザーがツールバーアイコンを押した時だけ対象タブへアクセスする
- 現在のメッセージ表示領域と交差する、可視状態の DOM 要素だけを一度走査する
- 候補をユーザーに提示し、明示的に選択されたものだけ保存する
- Discord の内部 API、ユーザートークン、Cookie、Gateway を使わない
- 自動スクロール、巡回、監視を行わない
- データを開発者のサーバーへ送信しない

案 1 の採用は Discord の許可や規約上の適用除外を意味しません。不確実性を許容できない場合は案 4 または案 5 を選びます。

## Consequences

### Positive

- 認証情報を扱わずに済む
- 対象とタイミングをユーザーが確認できる
- バックグラウンド収集を避けられる
- `activeTab`, `scripting`, `downloads` を中心とした比較的小さい権限で設計できる

### Negative

- 仮想スクロールのため、チャンネル履歴全体は取得できない
- Discord の DOM 変更に追随する保守が必要
- 埋め込みプレビューと添付を完全に区別できないケースがある
- URL の期限切れにより、一覧化後に保存が失敗する可能性がある
- 限定した方式でも規約上許可されるとは限らない

## Rejected alternatives

### 通常ユーザーの認証情報で API を呼ぶ

認証情報漏えいの影響が大きく、self-bot に該当する自動化につながるため採用しません。

### DOM の自動スクロール

網羅性は上がりますが、自動収集の性質、負荷、検出の不安定性、意図しない大量取得が増すため採用しません。

### 公式 Bot API のみを使う

Bot を追加できるサーバーでは、権限と API 契約が明確になる利点があります。一方、ユーザーが閲覧できても Bot を追加できないチャンネルには使えません。DOM 方式の適合性が確認できない場合の第一代替案とします。

### ブラウザ標準の一件保存

最も単純で権限追加がありませんが、複数ファイルの確認・保存という目的を満たしません。プロジェクト中止時の安全な代替手段です。

## Revisit triggers

- Discord から用途または方式に関する回答を得た
- Discord の利用規約や開発者ポリシーが変わった
- DOM 方式に必要な権限が設計より広くなった
- unpacked 配布では管理できない人数・更新頻度になった
- 公式 API で同じユーザー価値を実現できるようになった
