# ADR-0002: 少人数への unpacked 配布を継続する

- Status: Accepted
- Date: 2026-07-15
- Decision owners: Project owner

## Context

`0.2.0` の実装、自動検証、配布物検証、Chrome での手動確認が完了しました。次の運用形態として、少人数への unpacked 配布、組織の Chrome enterprise policy による管理、またはプロジェクト終了を選ぶ必要があります。

Chrome Web Store へ公開しないため、自動更新、Store 審査、一般利用者向けサポートはありません。一方、現在の利用規模では、配布対象者へ Project owner が直接連絡し、更新や利用停止を案内できます。

## Decision

少人数への unpacked 配布を継続します。次の条件をすべて維持します。

- 配布対象を、Project owner が直接連絡できる信頼済み利用者に限定する
- ZIP、SHA-256、CHANGELOG由来のGitHub Release本文を同じ連絡経路で渡す
- 最新の限定配布版だけをサポートし、更新は手動で行う
- 利用状況の遠隔測定、クラッシュ送信、自動更新確認を追加しない
- Chrome Web Store 公開と自己ホスト CRX 配布を行わない
- ADR-0001 の「表示中メディアの保存支援」という境界を維持する
- 権限追加、対応画面の拡大、自動処理の追加時は、実装前に ADR とセキュリティを再レビューする

配布人数の固定上限は設けません。ただし、Project owner が全利用者へ個別に更新・停止連絡できなくなった時点で、この判断を再検討します。

## Consequences

### Positive

- 配布対象と更新状況を把握できる
- 問題発生時に利用停止を直接案内できる
- 現在の最小権限とローカル処理だけの設計を維持できる

### Negative

- 利用者ごとにインストールと更新の手作業が必要
- 古いバージョンの利用を技術的に強制停止できない
- Store 審査、署名、自動更新を安全性の根拠にできない
- Project owner に配布、問い合わせ、DOM 変更追随の負担が集中する

## Rejected alternatives

### Chrome enterprise policy

現時点では組織管理された Chrome 環境を前提にしておらず、少人数の利用に対して運用が過大なため採用しません。組織配布が必要になった場合は再検討します。

### プロジェクト終了

限定用途での手動確認に問題がなく、保守可能な範囲で利用価値があるため、現時点では終了しません。終了条件に該当した場合は安全側へ倒して配布を停止します。

## Revisit triggers

- 全利用者へ直接更新・停止連絡できない規模になった
- 組織管理された端末への配布が必要になった
- 現在より広い権限、常駐処理、外部通信が必要になった
- Discord の規約、仕様、DOM が変わった
- 重大な誤取得、情報漏えい、または修正不能な不具合が発生した
- Project owner が継続保守できなくなった
