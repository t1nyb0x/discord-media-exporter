# プロダクト要件

## 1. 背景

Discord チャンネル内で閲覧できる画像や動画を、ユーザー自身が一件ずつ「名前を付けて保存」する作業は煩雑です。本プロジェクトは、現在ブラウザのメッセージ表示領域内に見えているメディアを一覧化し、ユーザーが内容を確認してローカルへ保存できる操作を提供します。

この文書の「表示中」は、メッセージ表示領域と要素の矩形が交差し、非表示状態ではないことを指します。DOM に存在していても、スクロール範囲外の要素は対象にしません。「取得」は、その表示中要素から既存のメディア URL と表示上のファイル名候補を読み取ることを指します。Discord の API から履歴を収集することは含みません。

## 2. 目的

- 現在開いているチャンネルの表示領域からメディア候補を検出する
- 保存前に、対象・件数・ファイル名をユーザーが確認できるようにする
- 選択したファイルを、衝突しにくく安全な名前でローカルへ保存する
- 認証情報やメッセージ本文を外部へ送信しない
- Discord や Chrome の仕様変更に対して、検出ロジックを交換・テストしやすくする

## 3. 対象ユーザー

- 自分が閲覧権限を持つチャンネルで、保存する権利または許可のあるメディアを整理したいユーザー
- 拡張機能の権限と保存対象を確認してから操作したいユーザー

閲覧できることと、複製・再配布できることは同義ではありません。保存するコンテンツの著作権、プライバシー、サーバールールの確認はユーザーの責任です。

## 4. MVP スコープ

### 含むもの

- `discord.com/channels/*` での手動スキャン
- 現在のメッセージ表示領域内に見えている次の候補の検出
  - 画像添付
  - 動画添付
  - 通常のファイル添付
- サムネイル、種類、推定ファイル名、重複状態の一覧表示
- 全選択、全解除、個別選択
- 選択した候補の一括ダウンロード開始
- 成功、失敗、スキップの件数表示
- 同じ URL の候補を一回だけ扱う重複排除
- UI を閉じても必要最小限のダウンロード状態を維持する

### 含まないもの

- 自動スクロール、ページ送り、チャンネル履歴の網羅的な収集
- Discord の内部・非公開 API、Gateway、ユーザートークン、Cookie の利用
- 通常ユーザーアカウントの自動操作（self-bot）
- バックグラウンドでの定期巡回や新着監視
- DM、スレッド、フォーラム、検索結果など全画面への初期対応
- 外部サーバーへのアップロード、クラウド同期、分析・広告 SDK
- 画面外にある、DOM へロード済みのメディア
- DRM 回避、期限切れ URL の復元、アクセス権の迂回
- メッセージ本文、投稿者一覧、リアクション等のエクスポート
- ZIP 化（MVP 後に、メモリ使用量を評価して判断）

## 5. ユーザーフロー

### 正常系

1. ユーザーは Discord Web の対象チャンネルを開く。
2. ユーザーは必要な範囲を自分で表示してから拡張機能を開く。
3. 拡張機能は対象ページであることを確認し、「この表示範囲をスキャン」を提示する。
4. ユーザー操作を受けて、コンテンツスクリプトが現在のメッセージ表示領域を一度だけ走査する。
5. 拡張機能は候補を一覧表示し、デフォルトでは全件未選択とする。
6. ユーザーは対象を選択して保存を実行する。
7. 保存先・重複時の挙動は Chrome の設定に従い、結果の要約を表示する。

### 例外系

- Discord 以外のページ: 対象外であることと、必要な操作を表示する
- チャンネル以外の Discord 画面: 未対応画面として扱う
- 候補がゼロ件: 表示範囲を確認する案内を出す。自動スクロールはしない
- URL が失効・403・ネットワークエラー: 対象名と理由を表示し、他の保存は継続する
- DOM 変更で解析不能: 解析失敗として通知し、診断情報はユーザーが明示的にコピーした場合だけ取得する
- 重複ファイル名: 一意なサフィックスを付けるか、Chrome の競合処理に委ねる

## 6. 機能要件

| ID    | 要件                                                             | 優先度 |
| ----- | ---------------------------------------------------------------- | ------ |
| FR-01 | ユーザーが操作した時だけ、現在のメッセージ表示領域をスキャンする | Must   |
| FR-02 | 候補ごとに URL、表示名、種別、取得元要素を正規化する             | Must   |
| FR-03 | URL のフラグメント除去後の値で重複を排除する                     | Must   |
| FR-04 | 保存対象を実行前に選択・確認できる                               | Must   |
| FR-05 | `chrome.downloads` で各ファイルを保存する                        | Must   |
| FR-06 | 成功・失敗・スキップを候補単位で追跡する                         | Must   |
| FR-07 | ファイル名からパス区切り、制御文字、予約名を除去する             | Must   |
| FR-08 | 設定として保存先サブディレクトリの命名規則を選べる               | Should |
| FR-09 | 画像・動画・その他添付で絞り込める                               | Should |
| FR-10 | 一覧にサムネイルを表示する                                       | Should |
| FR-11 | ユーザーが診断情報をクリップボードへコピーできる                 | Could  |
| FR-12 | DOM に存在しても表示領域外の候補を除外する                       | Must   |

## 7. 非機能要件

- セキュリティ: トークン、Cookie、Local Storage の認証情報を読まない
- プライバシー: メディア URL やメッセージ情報を開発者または第三者へ送信しない
- 最小権限: Discord 対象タブへの一時アクセスと `downloads` を基本とする
- 性能: 500 候補の走査・正規化を目安として 1 秒以内に完了する。ネットワーク転送時間は除く
- 安定性: 一件の失敗でバッチ全体を中断しない
- アクセシビリティ: キーボード操作、可視フォーカス、ラベル、十分なコントラストを提供する
- 保守性: DOM セレクターを UI・ダウンロード処理から分離し、fixture ベースでテストする
- 国際化: 初期 UI は日本語。文言をコードから分離し、将来の英語対応を妨げない

## 8. 受け入れ条件

- テスト用チャンネルに画像 2 件、動画 1 件、その他添付 1 件を表示した状態で、4 件が重複なく一覧に出る
- 2 件を選択すると、その 2 件だけがダウンロード開始される
- 同一 URL が複数の DOM 要素から見つかっても一覧は 1 件になる
- `../`, `/`, `\\`, 制御文字を含む表示名から安全なファイル名が生成される
- 対象外ページでは DOM を読まず、ダウンロードも開始しない
- 拡張機能コードとログに Discord の認証トークンが現れない
- ネットワーク遮断または 403 の候補があっても、他候補の処理が継続する
- 権限説明、プライバシー説明、規約上の注意、手動更新手順が配布物から到達できる

## 9. 未決事項

| 論点                               | 決定期限         | 決定方法                             |
| ---------------------------------- | ---------------- | ------------------------------------ |
| 限定用途での利用判断               | 第三者への提供前 | 利用規約を確認し、責任主体が承認     |
| unpacked 配布物の真正性確認        | MVP 検証前       | ハッシュとリリースノートの運用を決定 |
| 対応する Discord 画面範囲          | MVP 後           | 利用状況と fixture を基に決定        |
| ZIP 出力                           | MVP 後           | 大容量時のメモリと失敗復旧を計測     |
| ファイル名に投稿者・時刻を含めるか | 実装前           | プライバシーと識別性を比較           |
