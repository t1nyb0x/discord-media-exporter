# プロダクト要件

## 1. 背景

Discord チャンネル内で閲覧できる画像や動画を、ユーザー自身が一件ずつ「名前を付けて保存」する作業は煩雑です。本プロジェクトは、現在ブラウザのメッセージ表示領域内に見えているメディアを一覧化し、ユーザーが内容を確認してローカルへ保存できる操作を提供します。

この文書の「表示中」は、メッセージ表示領域と要素の矩形が交差し、非表示状態ではないことを指します。DOM に存在していても、スクロール範囲外の要素は対象にしません。「取得」は、その表示中要素から既存のメディア URL と表示上のファイル名候補を読み取ることを指します。Discord の API から履歴を収集することは含みません。

## 2. 目的

- 現在開いているチャンネルの表示領域からメディア候補を検出する
- 保存前に、対象・件数・ファイル名をユーザーが確認できるようにする
- 選択したファイルを、衝突しにくく安全な名前でローカルへ保存する
- 認証情報やメッセージ本文を外部へ送信しない
- Discord や Chrome の仕様変更に対して、検出ロジックを交換・テストしやすくする

## 3. 対象ユーザー

- 自分が閲覧権限を持つチャンネルで、保存する権利または許可のあるメディアを整理したいユーザー
- 拡張機能の権限と保存対象を確認してから操作したいユーザー

閲覧できることと、複製・再配布できることは同義ではありません。保存するコンテンツの著作権、プライバシー、サーバールールの確認はユーザーの責任です。

## 4. MVP スコープ

### 含むもの

- `discord.com/channels/*` でユーザーが開始する可視範囲の自動収集
- 現在のメッセージ表示領域内に見えている次の候補の検出
  - 画像添付
  - 動画添付
  - 通常のファイル添付
- サムネイル、種類、推定ファイル名、重複状態の一覧表示
- 全選択、全解除、個別選択
- 選択した候補の一括ダウンロード開始
- 成功、失敗、スキップの件数表示
- 同じ URL の候補を一回だけ扱う重複排除
- UI を閉じても必要最小限のダウンロード状態を維持する

### 含まないもの

- 無人の連続自動スクロール、ページ送り、チャンネル履歴の網羅的な収集
- Discord の内部・非公開 API、Gateway、ユーザートークン、Cookie の利用
- 通常ユーザーアカウントの自動操作（self-bot）
- バックグラウンドでの定期巡回や新着監視
- DM、スレッド、フォーラム、検索結果など全画面への初期対応
- 外部サーバーへのアップロード、クラウド同期、分析・広告 SDK
- 画面外にある、DOM へロード済みのメディア
- DRM 回避、期限切れ URL の復元、アクセス権の迂回
- メッセージ本文、投稿者一覧、リアクション等のエクスポート
- ZIP 化（Phase 5 の追加機能として扱い、MVP には含めない）

### Phase 5 で追加するもの

- 選択済み候補を一つのメディア ZIP としてローカル保存
- 個別保存と ZIP 保存の選択
- ZIP 生成の進捗、キャンセル、上限超過・失敗表示
- ZIP 利用時だけ要求する Discord CDN への任意ホスト権限

詳細な上限、失敗時の原子性、技術設計は[メディア ZIP 出力仕様](zip-export.md)を参照してください。

### Phase 6 で追加するもの

- ZIP固有の100件上限を廃止し、選択した候補を全件処理する
- 1ファイル50 MiB、入力合計100 MiBの固定上限を、OPFSクォータ・disk容量に基づく実行時判定へ置き換える
- ZIP64対応writerからOPFS一時ファイルへのbackpressure付きstreaming output
- 開始前の参考容量表示、quota・disk不足時の安全な失敗、一時ファイルcleanup
- Cache Storageへ入力responseを複製せず、archive全体をJavaScript heapへ保持しない

詳細は[Phase 6: 全選択候補のディスクストリーミングZIP](large-zip-export.md)と[ADR-0005](adr/0005-stream-large-zip-to-opfs.md)を参照してください。`0.5.0`でPhase 6を実装し、FR-17のPhase 5固定上限要件をFR-23からFR-27で置き換えます。大容量・容量不足・OS展開互換性のChrome Stable手動検証は別途リリースゲートとして残します。

### Phase 7 で追加するもの

- 任意の常時表示設定がOFFの場合は、popupを開いた明示操作後に停止状態の収集launcherを表示する
- 「Discordで開始ボタンを常に表示」を明示的にONにした場合は、Discordチャンネルへinactive launcherを自動表示する
- popupまたはページ内launcherの開始ボタンを押した後だけ、scanと表示変化の監視を開始する
- ユーザーが押すたびに、選択した方向へ表示高の80%だけ一回移動する
- 上端、下端、scroll container不明、500件到達時に無人で処理を継続しない
- 順序を重視する場合は、ユーザーが選んだ開始地点で収集結果を明示クリアし、一画面ずつ新しい投稿方向へ収集する
- ガイドまたはpopupから停止でき、チャンネル変更時に自動終了する
- timer・再帰・連続loopによる無人の自動スクロールを行わない
- 明示操作時だけ、現在表示中のスポイラーを解除できる

詳細は[Phase 7: ガイド付き一画面収集](guided-scroll-collection.md)と[ADR-0006](adr/0006-guide-one-scroll-step-per-user-action.md)を参照してください。

## 5. ユーザーフロー

### 正常系

1. ユーザーは Discord Web の対象チャンネルを開く。
2. ユーザーは必要な範囲を自分で表示してから拡張機能を開く。
3. 拡張機能は対象ページであることを確認し、popupとDiscord画面内のinactive launcherへ「自動収集を開始」を提示する。
4. ユーザーがpopupまたはページ内launcherの開始ボタンを押すと、コンテンツスクリプトが現在のメッセージ表示領域を走査し、表示変化の監視を開始する。
5. ユーザーがDiscordを自分でスクロールするか、画面内の「1画面戻る」を押すと、拡張機能は新しく表示された範囲を自動的に再走査する。
6. 拡張機能は同じチャンネルで表示された候補をセッション中に累積し、URL単位で重複を除外する。別チャンネルへ移動した場合は監視を停止し、候補を混在させない。
7. 拡張機能は候補を一覧表示し、デフォルトでは全件未選択とする。
8. ユーザーは必要に応じて自動収集を停止し、対象を選択して保存を実行する。収集結果は明示的にクリアできる。
9. 保存先・重複時の挙動は Chrome の設定に従い、結果の要約を表示する。

ZIP 保存を選んだ場合は、追加権限と推定一時空き容量を確認してから生成を開始し、選択項目がすべて成功した場合だけ一つの ZIP を保存します。

### 例外系

- Discord 以外のページ: 対象外であることと、必要な操作を表示する
- チャンネル以外の Discord 画面: 未対応画面として扱う
- 候補がゼロ件: 表示範囲を確認する案内を出す。自動スクロールはしない
- URL が失効・403・ネットワークエラー: 対象名と理由を表示し、他の保存は継続する
- DOM 変更で解析不能: 解析失敗として通知し、診断情報はユーザーが明示的にコピーした場合だけ取得する
- 重複ファイル名: 一意なサフィックスを付けるか、Chrome の競合処理に委ねる

## 7. 非機能要件

- セキュリティ: トークン、Cookie、Local Storage の認証情報を読まない
- プライバシー: メディア URL やメッセージ情報を開発者または第三者へ送信しない
- 最小権限: Discord 対象タブへの一時アクセスと `downloads` を基本とする
- 性能: 500 候補の走査・正規化を目安として 1 秒以内に完了する。ネットワーク転送時間は除く
- 安定性: 一件の失敗でバッチ全体を中断しない
- アクセシビリティ: キーボード操作、可視フォーカス、ラベル、十分なコントラストを提供する
- 保守性: DOM セレクターを UI・ダウンロード処理から分離し、fixture ベースでテストする
- 国際化: 日本語・英語UIを提供し、ChromeのUI言語が未対応の場合は英語へfallbackする。popup内の明示上書きをChrome言語より優先する
- ZIP 性能: 一件ずつ取得し、ZIP出力をOPFSへ逐次writeして、archiveサイズに比例したJavaScript heap増加を避ける
- ZIP 完全性: 選択項目の一部だけを含む ZIP を成功扱いせず、生成物の CRC と内容一致をテストする
- 収集上限: セッション中に保持する候補は1チャンネル最大500件とする

## 8. 受け入れ条件

- テスト用チャンネルに画像 2 件、動画 1 件、その他添付 1 件を表示した状態で、4 件が重複なく一覧に出る
- 2 件を選択すると、その 2 件だけがダウンロード開始される
- 同一 URL が複数の DOM 要素から見つかっても一覧は 1 件になる
- 自動収集開始後に同じチャンネルをスクロールすると、追加操作なしで新しく表示した候補が一覧へ追加される
- popupを開いただけでは候補をscanせず、ページ内launcherの開始操作後に初回候補が登録される
- popupとページ内launcherを続けて操作してもcollectorが一つだけ起動する
- 常時表示ONではDiscordチャンネルを開いた時点でinactive launcherが表示されるが、候補は収集されない
- 常時表示OFFまたは権限拒否後も、popupを開けばinactive launcherを表示できる
- 設定OFFまたはChrome側の権限取消後は、新しく開くDiscordチャンネルへlauncherを自動注入しない
- ポップアップを閉じて再度開いても同じチャンネルの収集結果が復元され、別チャンネルの候補とは混在しない
- `../`, `/`, `\\`, 制御文字を含む表示名から安全なファイル名が生成される
- 対象外ページでは DOM を読まず、ダウンロードも開始しない
- 拡張機能コードとログに Discord の認証トークンが現れない
- ネットワーク遮断または 403 の候補があっても、他候補の処理が継続する
- 権限説明、プライバシー説明、規約上の注意、手動更新手順が配布物から到達できる
- Chrome言語が日本語・英語・未対応言語の場合に想定localeを表示し、popupの三択変更がpopupと収集中ガイドへ即時反映される

Phase 5 の追加受け入れ条件:

- 画像 2 件、動画 1 件、その他添付 1 件を選択すると、同じ内容の 4 entry を持つ ZIP が一つだけ保存される
- 同名ファイルが複数ある場合も上書きされず、安全な一意名で全件が格納される
- 権限拒否、403、許可外 redirect、上限超過、キャンセルではメディア ZIP が保存されない
- popup を閉じて再度開いても、実行中または終了した ZIP の状態を確認できる
- ZIP 利用前は CDN の任意ホスト権限を要求せず、拒否後も個別保存を利用できる
- ZIP とログにチャンネル名、投稿者名、メッセージ本文、完全なメディア URL が追加されない

Phase 6 の追加受け入れ条件:

- 101件と500件をZIP固有の件数エラーなしで保存・展開できる
- 4 GiB超のarchiveをZIP64対応ツールで展開できる
- archiveサイズに比例してJavaScript heapが増加しない
- quota不足、disk不足、取得失敗、キャンセルで不完全ZIPを保存しない
- 入力responseのCache Storage複製がなく、終了後にOPFS一時ファイルが残らない
- checkboxの選択順にかかわらず、ZIP内ファイルが候補の取得順で`001_`から並ぶ
- 画像、動画、その他ファイルが混在しても、一回のscanで表示中DOMにある添付順で候補化される
- checkboxの選択順にかかわらず、個別保存とZIP保存が候補registryの順序を維持する
- OPFSの`File.type`が空でも完成archiveが`.zip`として保存される

添付順はbest-effortとします。Discordの仮想化DOMと可視範囲の境界により、同じメッセージの添付が複数回のscanへ分かれた場合や、互いに重ならない表示範囲を移動した場合は、元のメッセージ単位の添付順を復元せず、候補registryへの発見順を維持します。完全な順序を得るために画面外の要素、メッセージID、本文、投稿時刻、Discord内部APIを収集しません。

Phase 7 の追加受け入れ条件:

- popupを開いた明示操作後にinactive launcherを表示し、開始後だけactive操作を表示する
- 「1画面戻る」を一回押すと、古い投稿方向へ一回だけ移動する
- 「1画面進む」を一回押すと、新しい投稿方向へ一回だけ移動する
- 操作後に表示された添付だけを同じチャンネルの候補へ追加する
- 上端・下端で無人の再試行や連続scrollを開始しない
- 順序優先収集は、ユーザーが選んだ開始地点で明示クリアしてから下方向へ一画面ずつ進める
- 開始地点が最古投稿であることと、メッセージ単位の完全な添付順を保証しない
- ガイドの停止、popupの停止、別チャンネル移動でガイドが消える
- 500件到達時に追加移動を無効化する
- 「表示中のスポイラーを解除」で可視スポイラーだけが解除される
- 画面外・通常button・500件到達時はスポイラーを解除しない

## 9. 未決事項

| ---------------------------------- | ---------------- | ----------------------------------- |
| 限定用途での利用判断               | 第三者への提供前 | 利用規約を確認し、責任主体が承認    |
| unpacked 配布物の真正性確認        | MVP 検証前       | ハッシュとCHANGELOGの配布運用を決定 |
| 対応する Discord 画面範囲          | MVP 後           | 利用状況と fixture を基に決定       |
| ファイル名に投稿者・時刻を含めるか | 実装前           | プライバシーと識別性を比較          |
